## ShortX是什么

**这是什么？**

这是一个xposed模块。

主要功能包含一键指令和自动指令，类似于ColorOS的小布指令，或MIUI的自动任务，或者Tasker等自动任务应用。

相对其他同类软件，ShortX提供了更多系统级触发器与操作。



主要功能：

1. 一键指令：可由多个动作组合实现一项快捷功能，用户可以自主编排动作列表，随后只需要点击编排好的指令，即可实现用户自定义的一键操作。例如：一键发微博
2. 自动指令：可由触发器、条件、动作组合实现的自动化指令，用户可以指定某个触发器事件触发后，如果满足某个条件，则执行哪些动作。例如：用户可以配置在每天早晨8点，打开短信应用，向关心的人发一句早安问候。
3. 发现：用户可浏览、导入预置指令。

## 一键指令和自动指令



本文介绍一键指令和自动指令分别是什么。以及App中涉及到的几个概念。

**触发器**

触发器，顾名思义，是用来触发某个东西的，就好比赛跑的信号枪，信号就是比赛开始的触发器。

**条件**

条件，顾名思义，就是做事之前要具备的条件。

**动作**

动作，就是要做的事，例如，田径比赛的信号枪一响，就要跑步，跑步就是这里的动作。

**一键指令**

一键指令可以包含多个动作，可以通过组合简单的动作来实现复杂的功能。

**自动指令**

自动指令包含触发器、条件（可选）、动作，是指某个事件触发，且满足某个条件时（也可以无条件），要做什么事情。

这个概念对应到赛跑里，信号枪就是触发器，无条件，动作就是跑。



### 如何创建一键指令

本文档旨在帮助用户了解如何在我们的应用中创建一键指令，并将其放到桌面快捷方式和状态栏磁贴，以便实现自定义的快捷操作。

#### 创建一键指令

**步骤 1：进入一键指令设置**

1. 打开ShortX。
2. 寻找“一键指令”设置界面，通常可以在应用的主界面或设置中找到。

**步骤 2：创建新的一键指令**

1. 在“一键指令”设置界面中，点击“新增”或“添加”按钮。
2. 命名这个一键指令，以便在需要时可以轻松找到它。

**步骤 3：添加动作**

1. 在一键指令中，你可以添加多个动作。
2. 点击“新增动作”按钮来添加动作，这些动作可以修改全局变量、读取局部变量、执行某些操作等。

**步骤 4：保存和应用**

1. 完成一键指令的设置后，确保保存设置。
2. 一键指令将会在用户点击该指令时执行预定义的动作序列。

#### 将一键指令放到桌面快捷方式

**步骤 1：创建桌面快捷方式**

1. 打开ShortX。
2. 找到你创建好的一键指令。
3. 在该指令上，寻找一个选项来将这个指令创建为一个桌面快捷方式。

**步骤 2：使用桌面快捷方式**

1. 一旦将一键指令创建为桌面快捷方式，用户可以回到桌面，找到这个快捷方式。
2. 当用户点击这个快捷方式时，应用会按照预先设置的动作顺序执行操作。

#### 将一键指令放到状态栏磁贴

**步骤 1：创建状态栏磁贴**

1. 打开ShortX。
2. 找到你创建好的一键指令。
3. 在该指令上，寻找一个选项来将这个指令创建为一个状态栏磁贴。

**步骤 2：使用状态栏磁贴**

1. 一旦将一键指令创建为状态栏磁贴，用户可以在状态栏上找到这个磁贴。
2. 当用户点击这个磁贴时，应用会按照预先设置的动作顺序执行操作。

#### 总结

通过创建一键指令，并将其放到桌面快捷方式和状态栏磁贴，你可以实现自定义的快捷操作，以更便捷地执行预定义的一系列动作。请根据实际应用的界面和功能进行操作，为用户提供最佳的用户体验。

### 如何创建自动指令

应用提供了非常灵活和强大的自动指令功能，支持多个触发器、多个条件以及多个动作，这使得用户可以更加精细地配置自动化任务。以下是针对这些扩展功能的更新教程，指导用户如何使用多个触发器、多个条件以及多个动作：

**使用自动指令**

自动指令功能允许你创建多个触发器、多个条件以及多个动作，以便更精确地满足不同的自动化需求。

**步骤 1：进入自动指令设置**

首先，打开你的Xposed模块应用，找到自动指令界面。通常，你可以在应用的主界面或设置中找到这个选项。

**步骤 2：创建新的自动指令**

在自动指令设置界面中，你应该能够找到一个“新增”或“添加”按钮，用来创建一个新的自动指令。点击这个按钮。

**步骤 3：设置触发器**

在创建自动指令时，你可以选择添加多个触发器。这意味着任何一个触发器的事件都能触发该指令。你可以为每个触发器选择不同的事件类型，如“时间”、“位置”、“通知”等。

**步骤 4：设置条件**

一旦触发器被设置，你可以进一步设置多个条件。在条件设置界面，你应该能够选择条件类型（Any或All）。如果选择“Any”，只需要满足任意一个条件即可触发动作。如果选择“All”，所有条件必须同时满足才能触发动作。你可以设置多个条件来进一步精确控制触发条件。

**步骤 5：添加动作**

在自动指令中，你可以添加多个动作。这些动作会按照你的设置顺序执行。点击“新增动作”按钮来添加动作，这些动作可以修改全局变量、读取局部变量、执行某些操作等。

**步骤 6：保存和应用**

完成自动指令的设置后，确保保存设置。自动指令将会在满足任意一个触发器事件且满足指定条件时启动，并按照顺序执行你指定的动作。

通过支持多个触发器、多个条件以及多个动作，你的应用可以为用户提供高度灵活的自动化功能，满足不同情景下的自动化需求。

## 指令配置 - 重复策略

### 重复策略

当涉及到自动指令的 "重复执行策略" 时，用户可以根据需要来设置当同一条指令在执行中，再次被触发时，对动作任务的处理方式。这种策略可以确保在自动指令执行期间避免不必要的冲突或混淆。以下是一个关于如何设置 "重复执行策略" 的介绍文档和使用说明：

---

#### 自动指令重复执行策略设置

**介绍**

"重复执行策略" 是我们应用中一个重要的功能，允许用户自定义当同一条自动指令在执行中再次被触发时，对动作任务的处理方式。这种策略可以帮助用户更好地管理自动化任务，避免冲突并确保按照用户的预期执行。

#### **设置重复执行策略**

在创建或编辑自动指令时，你可以找到 "重复执行策略" 设置选项。以下是该选项的三种可选策略：

1. **取消旧任务执行新任务**

选择此策略后，如果同一条指令正在执行中，当再次触发这条指令时，会取消旧任务并立即执行新任务。这个选项适用于用户希望保持最新指令状态，而不关心旧任务的情况。

2. **取消新任务，保持旧任务**

这个策略允许用户在同一条指令执行中时，保持旧任务，而取消触发的新任务。这对于某些需要确保旧任务完整执行的场景很有用。

3. **新任务和旧任务同时执行**

选择此策略后，如果同一条指令正在执行中，当再次触发这条指令时，新任务会和旧任务同时执行。这个选项适用于需要同时处理多个任务的情况。

#### 使用说明

1. 打开ShortX，并进入自动指令设置界面。
2. 创建或编辑一条自动指令。
3. 在设置界面中，寻找 "重复执行策略" 选项。
4. 选择你需要的执行策略：取消旧任务执行新任务、取消新任务保持旧任务、或新任务和旧任务同时执行。
5. 确保保存设置。

#### 注意事项

- 根据你的实际需求，选择适合的重复执行策略。
- 需要考虑不同策略对任务执行顺序和冲突的影响。
- 请确保用户明确了解每种策略的含义和预期效果，以避免混淆和误操作。

通过设置合适的重复执行策略，你可以更好地管理自动化任务，确保指令按照你的预期执行。

## 指令配置 - 同步策略

在我们的应用中，你可以设置指令的执行策略，以控制指令内多个动作的顺序执行以及不同指令之间的执行方式。以下是我们支持的三种指令执行策略：

**异步执行**

异步执行是指不同指令的动作可以并行执行，不会互相等待。这意味着多个指令可以在同一时间内同时执行，从而提高效率。这个选项适用于那些独立执行的指令，无需等待其他指令的完成。

**同步执行（全局）**

同步执行是指不同指令的动作在全局范围内按照先后顺序依次执行。这意味着两个指令将按照设置的顺序逐个执行，确保执行的有序性。然而，需要注意的是，由于是全局性质的同步执行，不要在同步指令内执行耗时动作，否则可能会导致其他指令被阻塞。

**同指令同步执行**

同指令同步执行是一个特殊的同步模式，其作用域限定在同一条指令内部。我们为每条指令都分配了一个专属的执行器，确保相同指令内的多个动作按照顺序依次执行，不同指令之间则异步执行。这个选项适用于需要在同一指令内保持顺序执行的场景。



#### 使用说明

1. 打开ShortX，并进入指令设置界面。
2. 创建或编辑一条指令。
3. 在设置界面中，寻找 "执行策略" 或 "同步模式" 选项。
4. 根据你的需求，选择适合的执行策略：异步执行、全局同步执行、或同指令同步执行。
5. 确保保存设置。
6. 

#### 注意事项

- 对于同步执行选项（全局或同指令），请避免在指令内执行耗时的操作，以免阻塞其他指令的执行。
- 理解每种执行策略的含义和影响，确保根据实际需求进行配置，以获得最佳的执行效果。

通过合理设置指令的执行策略，你可以更好地控制指令的执行顺序和方式，以满足不同的自动化需求。

## 指令配置 - Hooks

"ShortX" 应用中支持使用 Hooks 来在指令开启、关闭、删除时执行不同的动作，这为用户提供了在一键指令生命周期中进行初始化和清理工作的灵活性。下面是关于如何使用这些 Hooks 的详细说明：

---

#### 使用指令 Hooks

"ShortX" 应用支持在指令的不同生命周期阶段执行不同的动作，这些特殊的动作称为 Hooks。用户可以利用这些 Hooks 去进行一些初始化和清理工作，以确保指令的有效执行和管理。

1. **指令开启时的 Hooks**

当用户开启一条指令时，可以指定在这个阶段执行特定的动作。这对于需要在指令执行前准备一些数据、设置一些状态的场景非常有用。

2. **指令关闭时的 Hooks**

在用户关闭一条指令时，可以设置在指令关闭的过程中执行特定的动作。这对于需要在指令执行结束后进行一些收尾工作的情况非常实用。

3. **指令删除时的 Hooks**

当用户删除一条指令时，可以设置在指令被删除时执行特定的动作。这对于需要在指令删除后清理相关数据、状态的情况非常有帮助。

#### 使用说明

1. 打开 "ShortX" 应用，并进入指令设置界面。
2. 创建或编辑一条一键指令。
3. 在设置界面中，寻找 "指令 Hooks" 或 "生命周期动作" 选项。
4. 在对应的选项下，设置在指令开启、关闭、删除时需要执行的动作，可以是内置的动作，也可以是用户自定义的操作。

#### 注意事项

- 请确保你理解每个 Hook 的触发时机和执行动作，以确保其在指令的生命周期中发挥正确的作用。
- 合理设置 Hooks 可以保证指令的执行和清理工作在适当的时机完成。

#### Hooks 的应用场景

- 在指令开启时，执行一些初始化操作，确保指令执行的前提条件满足。
- 在指令关闭时，执行一些清理工作，释放资源或还原状态。
- 在指令删除时，清除与指令相关的数据，确保删除操作完整。

通过合理设置指令 Hooks，用户可以更好地管理指令的生命周期，确保指令的正常执行和清理工作。

> 全局变量是一种在程序中定义的变量，其作用范围不仅限于特定的函数、方法或代码块，而是在整个程序中都可以访问和使用的变量。具体来说，全局变量在程序的各个部分都可以读取和修改，无论这些部分位于何处。全局变量的主要特点如下：
>
> 1. **作用范围广**: 全局变量在程序中的各个函数、方法和代码块中都可以被访问，无需通过参数传递或其他特殊的传递机制。
>
> 2. **持久性**: 全局变量的生命周期延伸到整个程序的运行期间，它会在程序启动时被创建，直到程序终止才会被销毁。因此，可以在不同的函数或方法中共享数据。
>
> 3. **易于修改**: 由于全局变量在程序的各个地方都可以修改，所以需要特别小心以确保数据的一致性和正确性。一个函数修改了全局变量的值可能会影响到其他部分的行为。
>
> 4. **命名空间**: 全局变量存在于全局命名空间中，因此需要确保全局变量的名称不与其他部分冲突，以避免命名冲突。
>
> 使用全局变量可能会带来一些挑战，因为它们可能导致代码变得难以理解、调试和维护。在编程中，通常会鼓励尽量减少全局变量的使用，而是使用局部变量、参数传递或其他封装机制来限制变量的作用范围，从而提高代码的可读性和可维护性。

### 局部变量

什么是局部变量？

> 局部变量是在程序的特定作用域内定义的变量，它们只能在这个作用域内被访问和使用，超出该作用域就无法访问。局部变量的主要特点如下：
>
> 1. **作用范围有限**: 局部变量仅在定义它们的函数、方法或代码块内部有效。它们不能被在其他函数、方法或代码块中直接访问，也不会与其他作用域中的同名变量发生冲突。
>
> 2. **生命周期短暂**: 局部变量的生命周期与其所在的作用域相关。当函数、方法或代码块执行完毕时，局部变量会被销毁，其占用的内存会被释放，这也有助于内存管理。
>
> 3. **隔离性**: 局部变量的隔离性使得每个作用域都可以拥有独立的变量，不会受到外部作用域中同名变量的影响，从而有助于避免命名冲突和数据混乱。
>
> 4. **局部性原则**: 将变量的作用范围限制在需要的最小范围内有助于提高代码的可读性、可维护性和安全性。这也是良好编程实践的一部分。
>
> 局部变量通常用于存储临时性的数据，如函数的参数、临时计算结果或其他仅在特定代码块中需要使用的数据。在一些编程语言中，局部变量的定义方式和作用范围都受到严格控制，以确保代码的健壮性和一致性。

首先，打开你的Xposed模块应用，找到指令编辑界面。这通常是在设置中的“指令”或“动作”选项。

**步骤 2：选择动作**

在指令编辑界面，你可以为指令添加多个动作，这些动作将会按照你的设置依次执行。在需要使用局部变量的动作上点击编辑。

**步骤 4：修改和读取局部变量**

在同一个动作内，你可以使用局部变量来存储和传递数据。当需要修改局部变量的值时，你可以在动作的设置中找到局部变量的选项，然后更新它的值。当需要读取局部变量的值时，同样在动作的设置中找到局部变量的选项或者直接使用`{localVarOf$变量名}`来引用，将其作为参数使用。

**步骤 5：保存和应用**

完成动作的编辑后，确保保存设置。当你触发这个指令时，相应的局部变量将会在动作的执行过程中使用。请注意，局部变量的生命周期仅限于当前指令的执行过程中，指令执行完毕后局部变量的值会被清除。

- **获取外部数据**：可以使用 "Http请求" 动作获取远程服务器的数据。

#### 使用 "全局复制" 动作

1. 打开 "ShortX" 应用，进入指令的创建或编辑界面。
2. 在动作列表中选择 "全局复制" 动作。

#### 执行 "全局复制" 动作的步骤

1. 当 "全局复制" 动作被触发时，你会看到屏幕上出现一个紫色的边框，这是提示你可以点击页面上的任意文本。
2. 点击任意文本后，会弹出一个对话框，其中包含了已复制的文本内容。
3. 在对话框中，你可以随时编辑文本，修改为你需要的内容。
4. 点击复制按钮，已编辑的文本会被复制到剪贴板，方便你在其他地方粘贴使用。

#### 应用场景

- **快速复制和编辑**："全局复制" 动作可以帮助你快速复制页面上的任意文本，并在对话框中进行编辑和调整。
- **提高效率**：无需手动选择文本，这个功能让文本操作更快速、便捷。



通过 "ShortX" 应用的 "全局复制" 动作功能，用户可以轻松地复制页面上的文本，并在对话框中进行编辑，实现文本的定制化和修改，提高工作效率和方便性。

## 常用MVEL表达式


MVEL表达式能实现很多功能，因为它可以调用Android SDK，而且ShortX还为MVEL执行器添加了Android Context，有了这两个条件，它能实现的功能就非常多了。

MVEL表达式[语法官方网站](http://mvel.documentnode.com/#language-guide-for-2.0)
Andrid SDK [API 文档](https://developer.android.com/reference/packages)


#### Webview 相关

**获取当前webview实现的包名**

```java
android.webkit.WebViewFactory.getUpdateService().getCurrentWebViewPackageName()
```

**判断当前webview实现的包名是否是XXX**

```java
android.webkit.WebViewFactory.getUpdateService().getCurrentWebViewPackageName() == "com.google.android.webview"
```


**切换webview**

```java
android.webkit.WebViewFactory.getUpdateService().changeProviderAndSetting("package_name")
```



#### 系统设置相关

**获取字体缩放**

```java
android.provider.Settings$System.getFloat(context.contentResolver, "font_scale")
```

**设置字体缩放**

默认是1.0f，表示缩放倍数

```java
android.provider.Settings$System.putFloat(context.contentResolver, "font_scale", 1.0)
```

**获取Wifi密码**

```java
wifiPasswords = "";

configs = context.getSystemService(context.WIFI_SERVICE).getPrivilegedConfiguredNetworks();
    
foreach(conf : configs) {
    wifiPasswords += conf.SSID + "-->" + conf.preSharedKey + "\n";
}
          
wifiPasswords;
```
